About This Project

Bedrock Server is a comprehensive open-source toolkit for running and managing Minecraft: Bedrock Edition servers. It simplifies setup, enhances stability, and gives server owners fine-grained control over their multiplayer environments with automation, mod/plugin support, and a web control panel.

  • Purpose: Make Bedrock server hosting accessible, stable, and customizable for everyone.
  • Approach: Bash/Python/Node.js scripts, Docker, FastAPI backend, React web frontend, and robust documentation.
  • Community: Contributions, ideas, and support are welcome! Join us on GitHub.

Getting Started

  1. Download the Minecraft: Bedrock Edition server binary.
  2. Install Git and (optionally) Docker.
  3. Clone the repository: 
    git clone https://github.com/Frank1o3/bedrock-server.git
cd bedrock-server
  4. Follow the setup instructions in the README and scripts.

Usage

Run the server using the included scripts or via Docker Compose: 

./start.sh

By default, the server launches with provided configs, stores world data in worlds/, and logs output to logs/. Modify config files to customize gameplay, permissions, or advanced settings.

  • Manual scripts: ./start.sh, ./backup.sh, ./rollback.sh
  • Docker Compose: 
    docker compose -f docker-compose.yml build
docker compose -f docker-compose.yml up -d

Architecture

Three-Tier System

  1. Game Server Layer:
    • Minecraft Bedrock server executable, world data, configs, automatic backups.
  2. API Layer:
    • Python FastAPI backend, route handlers, SQLite-based user management, mod loader.
  3. Frontend Layer:
    • React SPA for real-time logs, mod/config management, authentication, and server control.

Key Features

  • Real-time log streaming (WebSockets) for server and bash terminal.
  • Role-based authentication and access control.
  • Automated scheduled and manual backups with rollback.
  • Mod management (upload/enable/disable/remove, world config updates).
  • Single-container deployment with multiple orchestrated processes.
  • CI/CD pipeline for Docker images.
